Problem Statement:

ROS development contains high barriers of
entry for new developers and environments.

Problem Characteristics


  • Each ROS node has at least one terminal window associated with it.
    Best case scenario: n Nodes = n Terminals

  • Terminal windows do not automatically have distinct attributes to easily identify one from another.

  • Navigating multiple windows can stall process flow.

  • There is no distinction when changes in one node affects a related node.

  • The ROS environment doesn't provide a GUI for folders, making it difficult to browse code.

  • Simple debugging methodologies become time consuming due to how ROS displays errors.

  • The combination these characteristics increase the already steep learning curve of ROS.
