Problem Statement:

Robot Operating System (ROS) requires an
abundance of terminal windows for building robot
applications. Developing robotics applications is
inefficient with the absence of customizable
terminal window settings.

Problem Characteristics


  • Each node needs multiple terminals, sometimes leading to 30 plus windows open at once.

  • Terminal windows do not automatically have distinct attributes to easily identify one from another.

  • Navigating multiple windows can stall process flow.

  • Programming on x+1 terminals, with no distinction of where an input creates an output, is time consuming.
